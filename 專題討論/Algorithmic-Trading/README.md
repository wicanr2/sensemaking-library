# Algorithmic Trading 專題討論

這個專題整理演算法交易、訂單拆分、市場微結構與看盤資料解讀。重點不是提供交易建議，而是把「盤面上看到的成交、單量、價格與速度」拆回背後可能的執行機制。

## 文章

- [VWAP 與 TWAP：大單如何被拆成市場看起來普通的小單](./01-vwap-twap-order-splitting.md)

## 目前核心問題

1. 為什麼大機構不直接用一張大單買賣？
2. VWAP / TWAP 這類演算法到底在最佳化什麼？
3. 看盤軟體的「大單 / 小單」分類為什麼可能誤導？
4. 黑池、內部化、冰山單與演算法拆單，各自在哪一層隱藏資訊？
5. 散戶能觀察什麼，不能過度推論什麼？

## 使用提醒

- 這裡討論的是市場微結構與執行邏輯，不是投資建議。
- 單一看盤指標很容易被交易拆單、撮合場域與資料分類方式扭曲；更可靠的方式是同時看價格、成交量、時間、盤口變化與後續延續性。
